2. Creating a Discord Application

To use CubPresence, you need to create a Discord Application. This is free and takes less than a minute.

  1. Open the Discord Developer Portal
    Go to discord.com/developers/applications and sign in with your Discord account.
  2. Create a New Application
    Click the "New Application" button in the top right. Give your application a name - this will be the activity title shown on your profile (e.g., "Playing Your App Name").
  3. Copy Your Application ID
    On the application's page, you'll see an "Application ID" (also called Client ID). Copy this number - you'll paste it into CubPresence.
  4. Add Images (Optional)
    Go to "Rich Presence" > "Art Assets" in the sidebar. Upload images and give them names (keys). You can use these names in CubPresence's image fields.
Tip: You can create multiple Discord applications with different names to switch between different activities!

3. Activity Types

Discord supports different activity types that change how your presence is displayed:

Type Display Example
Playing Playing [App Name] Playing Minecraft
Streaming Streaming [App Name] Streaming on Twitch
Listening to Listening to [App Name] Listening to Spotify
Watching Watching [App Name] Watching YouTube
Competing in Competing in [App Name] Competing in Tournament
Note: The "Streaming" type may show a purple "LIVE" badge on your profile. "Listening to" is what Spotify uses for its integration.

4. Timestamps

Timestamps show elapsed time or countdowns on your presence. CubPresence offers several timestamp options:

Available Options

  • None - No timestamp displayed
  • Since Update - Time since the presence was last updated
  • Since Connect - Time since you connected to Discord
  • Since App Start - Time since CubPresence was opened
  • Local Time - Shows your local time today
  • Custom - Set a specific start/end date and time

Custom Timestamps

With custom timestamps, you can:

  • Set only a start time to show elapsed time from that moment
  • Set both start and end times to show a countdown

5. Images

You can display two images on your presence: a large image and a small image that overlays the corner of the large one.

Using Image URLs

The easiest way is to use direct image URLs:

  • Paste any https:// image URL directly into the image field
  • Supported formats: PNG, JPG, GIF
  • Recommended size: 512x512 pixels or larger

Using Discord Assets

You can also upload images to your Discord application:

  1. Go to your app in the Developer Portal
  2. Navigate to "Rich Presence" > "Art Assets"
  3. Upload images and give them names
  4. Use the image name (key) in CubPresence
Tip: Image tooltips appear when someone hovers over your images. Use them to add extra information!

6. Buttons

Add up to 2 clickable buttons to your presence that link to any URL.

Button Guidelines

  • Each button needs a label (max 32 characters) and a URL
  • URLs must start with https:// or http://
  • Common uses: website links, social media, join links, donation pages
Important: You cannot see or click your own buttons! They only appear for others viewing your profile. Ask a friend to verify they work.

8. Settings

CubPresence includes several helpful settings:

Startup Options

  • Run on system startup - Launch CubPresence when Windows starts
  • Start minimized to tray - Start hidden in the system tray
  • Auto-connect on startup - Automatically connect to Discord when opened

Behavior Options

  • Minimize to tray on close - Keep running in the background when you click X
  • Show notifications - Desktop notifications for connection status
  • Remember presence on close - Save your settings and restore them next time

System Tray

CubPresence runs in your system tray. Right-click the tray icon for quick actions:

  • Open the main window
  • Quick Connect/Disconnect
  • Check for updates
  • Quit the application

9. Troubleshooting

"Failed to connect to Discord"

  • Make sure Discord is running on your computer
  • Try restarting Discord
  • Make sure you're not running Discord in a browser - the desktop app is required

"Application ID should be a number"

  • The Application ID is a long number (e.g., 1234567890123456789)
  • Don't confuse it with the Public Key or other values
  • Copy it directly from the Developer Portal

Images not showing

  • If using URLs, make sure they're direct image links (ending in .png, .jpg, etc.)
  • If using Discord assets, make sure the asset name matches exactly
  • New Discord assets can take up to 10 minutes to become available

Buttons not appearing

  • Remember: you can't see your own buttons!
  • Make sure both label and URL are filled in
  • URLs must start with http:// or https://

Party size not showing

  • Both Party Size and Party Max must be filled in
  • Party Size cannot be larger than Party Max
